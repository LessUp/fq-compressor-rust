# Release v0.1.0

**Release Date**: 2026-03-07

## Release Notes

🎉 **First stable release of fqc** — a high-performance FASTQ compressor in Rust.

This is a complete port of the [C++ fq-compressor](https://github.com/LessUp/fq-compressor) with feature parity, sharing the same `.fqc` archive format.

---

## Highlights

### 🧬 Compression Algorithms

#### ABC Algorithm (Alignment-Based Compression)
- **Use case**: Short reads (< 300bp)
- **Method**: Consensus + delta encoding
- **Benefits**: High compression ratio by exploiting sequence similarity

#### Zstd Compression
- **Use case**: Medium (300bp – 10kbp) and long reads (> 10kbp)
- **Method**: Length-prefixed encoding followed by Zstd compression
- **Benefits**: Fast, general-purpose compression with configurable levels (1-19)

#### SCM Quality Compression
- **Use case**: Quality score compression
- **Method**: Statistical Context Model with arithmetic coding
- **Variants**: Order-2 for short/medium reads, Order-1 for long reads

### 📊 Processing Modes

| Mode | Description | Best For |
|------|-------------|----------|
| **Default** | Batch processing with global minimizer-based reordering | Standard compression |
| **Streaming** | Low-memory stdin compression without reordering | Pipes, memory-constrained |
| **Pipeline** | 3-stage Reader→Compressor→Writer with backpressure | Maximum throughput |

### 💾 I/O Features

- **Async I/O** — Background prefetch and write-behind buffering
- **Compressed Input** — Transparent decompression of `.gz`, `.bz2`, `.xz`, `.zst`
- **Random Access** — Block-indexed archive for partial decompression
- **Range Extraction** — Extract specific read ranges (`--range 1:1000`)

### 🧬 Paired-End Support

- Separate file input (`-i R1.fastq -2 R2.fastq`)
- Interleaved file input (`--interleaved`)
- PE layout options (interleaved/consecutive storage)
- Split output on decompress (`--split-pe`)

### ⚙️ Quality Modes

| Mode | Description | Compression Impact |
|------|-------------|-------------------|
| **Lossless** | Exact quality preservation | Baseline |
| **Illumina8Bin** | 8-bin quantization | ~30% improvement |
| **Discard** | Replace all with `!` (Phred 0) | Maximum compression |

### 🔧 CLI Commands

| Command | Description |
|---------|-------------|
| `fqc compress` | Compress FASTQ to FQC |
| `fqc decompress` | Decompress FQC to FASTQ |
| `fqc info` | Display archive information (text/JSON) |
| `fqc verify` | Verify archive integrity |

### 🚦 Exit Codes

| Code | Name | Description |
|------|------|-------------|
| 0 | Success | Operation completed |
| 1 | Usage | Invalid arguments |
| 2 | IoError | I/O error |
| 3 | FormatError | Invalid format |
| 4 | ChecksumError | Integrity violation |
| 5 | Unsupported | Unsupported codec/version |

## Getting Started

```bash
# Install
cargo install fqc

# Or download from releases
# https://github.com/LessUp/fq-compressor-rust/releases/tag/v0.1.0

# Compress
fqc compress -i reads.fastq -o reads.fqc

# Decompress
fqc decompress -i reads.fqc -o reads.fastq

# Info
fqc info -i reads.fqc

# Verify
fqc verify -i reads.fqc
```

---

## Migration from C++ version

The `.fqc` format is compatible between Rust and C++ implementations. Archives created with either can be decompressed with the other.
