# 版本 v0.1.0

**发布日期**: 2026-03-07

## 发布说明

🎉 **fqc 首个稳定版本** — 用 Rust 编写的高性能 FASTQ 压缩器。

这是 [C++ fq-compressor](https://github.com/LessUp/fq-compressor) 的完整移植版本，功能对等，共享相同的 `.fqc` 归档格式。

---

## 核心亮点

### 🧬 压缩算法

#### ABC 算法（基于对齐的压缩）
- **适用场景**: 短读段 (< 300bp)
- **方法**: 共识序列 + 增量编码
- **优势**: 利用序列相似性实现高压缩比

#### Zstd 压缩
- **适用场景**: 中等 (300bp – 10kbp) 和长读段 (> 10kbp)
- **方法**: 长度前缀编码后接 Zstd 压缩
- **优势**: 快速、通用的压缩，支持可调级别 (1-19)

#### SCM 质量值压缩
- **适用场景**: 质量值压缩
- **方法**: 统计上下文模型 + 算术编码
- **变体**: 短/中等读段使用二阶模型，长读段使用一阶模型

### 📊 处理模式

| 模式 | 描述 | 最佳场景 |
|------|-------------|----------|
| **默认模式** | 批量处理，全局 minimizer 重排序 | 标准压缩 |
| **流式模式** | 低内存 stdin 压缩，无重排序 | 管道、内存受限 |
| **流水线模式** | 3 阶段读取器→压缩器→写入器，带背压 | 最大吞吐 |

### 💾 I/O 特性

- **异步 I/O** — 后台预取和写后缓冲
- **压缩输入** — 透明解压 `.gz`、`.bz2`、`.xz`、`.zst`
- **随机访问** — 块索引归档支持部分解压
- **范围提取** — 提取特定读段范围 (`--range 1:1000`)

### 🧬 配对端支持

- 分离文件输入 (`-i R1.fastq -2 R2.fastq`)
- 交错文件输入 (`--interleaved`)
- PE 布局选项（交错式/连续式存储）
- 解压时分离输出 (`--split-pe`)

### ⚙️ 质量模式

| 模式 | 描述 | 压缩影响 |
|------|-------------|-------------------|
| **无损模式** | 精确保留质量值 | 基准 |
| **Illumina8Bin** | 8 箱量化 | 提升约 30% |
| **丢弃模式** | 全部替换为 `!` (Phred 0) | 最大压缩 |

### 🔧 CLI 命令

| 命令 | 描述 |
|---------|-------------|
| `fqc compress` | 压缩 FASTQ 到 FQC |
| `fqc decompress` | 解压 FQC 到 FASTQ |
| `fqc info` | 显示归档信息（文本/JSON） |
| `fqc verify` | 验证归档完整性 |

### 🚦 退出码

| 代码 | 名称 | 描述 |
|------|------|-------------|
| 0 | Success | 操作成功完成 |
| 1 | Usage | 无效参数 |
| 2 | IoError | I/O 错误 |
| 3 | FormatError | 无效格式 |
| 4 | ChecksumError | 完整性违规 |
| 5 | Unsupported | 不支持编解码器/版本 |

## 快速开始

```bash
# 安装
cargo install fqc

# 或从 releases 下载
# https://github.com/LessUp/fq-compressor-rust/releases/tag/v0.1.0

# 压缩
fqc compress -i reads.fastq -o reads.fqc

# 解压
fqc decompress -i reads.fqc -o reads.fastq

# 查看信息
fqc info -i reads.fqc

# 验证
fqc verify -i reads.fqc
```

## 从 C++ 版本迁移
