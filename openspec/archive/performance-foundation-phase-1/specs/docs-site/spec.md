# docs-site change

## ADDED Requirements

### Requirement: architecture docs must carry a maintained performance summary

The docs site MUST expose a concise architecture-level summary of the current performance bottlenecks, recommended direction, active phase scope, and deferred follow-up.

#### Scenario: a reader wants the current performance plan

- **WHEN** they browse the architecture section
- **THEN** they can open a maintained performance roadmap page from site navigation
- **AND** the page summarizes the current direction without copying a full research report into the docs site
