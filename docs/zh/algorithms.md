# 压缩算法详解

> 另请参阅：[architecture.md](architecture.md)（模块架构）、[format-spec.md](format-spec.md)（二进制格式规范）

## 概述

fqc 根据读段长度分类选择不同的压缩策略：

| 读段长度 | 序列编码器 | 质量值编码器 | 全局重排序 |
|----------|------------|--------------|------------|
| 短读段 (< 300bp) | ABC（共识 + 增量） | SCM 二阶模型 | ✓ 是 (minimizer) |
| 中等读段 (300bp – 10kbp) | Zstd | SCM 二阶模型 | ✗ 否 |
| 长读段 (> 10kbp) | Zstd | SCM 一阶模型 | ✗ 否 |

算法实现位于 `src/algo/` 目录：

| 模块 | 职责 |
|------|------|
| `block_compressor.rs` | 块级 ABC/Zstd 压缩与解压 |
| `global_analyzer.rs` | Minimizer 提取与全局排序 |
| `quality_compressor.rs` | SCM 算术编码质量值压缩 |
| `id_compressor.rs` | 读段 ID 压缩 |
| `pe_optimizer.rs` | 配对端反向互补优化 |

## ABC 算法（基于对齐的压缩）

ABC 算法用于短读段（如 Illumina），利用序列间的高度相似性进行共识 + 增量编码。

### 处理流程

```
读段 → 全局排序 → 块分割 → 每块处理:
  1. 构建 Contig（共识序列 + 对齐）
  2. 增量编码（仅存储差异）
  3. 序列化 + Zstd 压缩
```

### 步骤 1：共识序列构建

每个块内，读段被聚类为 **contig**（对齐读段簇）：

1. 选取未分配的读段作为种子，用其序列初始化共识
2. 对每个剩余读段，在 `[-max_shift, +max_shift]` 范围内尝试对齐（正向 + 反向互补）
3. 若 Hamming 距离 ≤ 阈值，将读段加入 contig，更新碱基频率计数
4. 所有读段处理完成后，以每个位置的多数碱基重新计算最终共识序列

**共识序列**以每个位置的四碱基频率 `[A, C, G, T]` 存储，最终取频率最高的碱基。

### 步骤 2：增量编码

Contig 中每条读段相对于最终共识序列进行增量编码：

| 字段 | 类型 | 说明 |
|------|------|------|
| `position_offset` | i16 | 相对共识序列的对齐偏移量 |
| `is_rc` | bool | 是否反向互补 |
| `mismatch_positions` | Vec\<u16\> | 与共识不同的位置索引 |
| `mismatch_chars` | Vec\<u8\> | 编码后的变异碱基 |

变异碱基编码规则：
- 重叠区域内的变异：使用噪声编码（类似 XOR）
- 共识范围外的延伸：存储原始碱基

### 步骤 3：序列化

每个 contig 序列化为以下结构：

1. 共识序列（长度前缀）
2. 增量读段数量
3. 每个增量读段：original_order, offset, is_rc, read_length, mismatch count, positions, chars

整个数据块最终使用 Zstd 压缩。

### 负偏移处理

当 `shift < 0` 时，读段延伸到共识序列起始位置之前：

- `cons_start = 0`, `read_start = |shift|`
- 重叠前的碱基（位置 `0..read_start`）作为原始碱基存入 mismatch 数据
- 解压时直接恢复（不经过噪声解码）

## 全局读段排序

对于短读段，minimizer 排序将相似读段聚集在一起，显著提升 ABC 算法的压缩比。

实现位于 `src/algo/global_analyzer.rs`。

### 算法

1. 从每条读段提取**规范 k-mer minimizer**（正向与反向互补中较小者）
2. 按 minimizer 值排序所有读段
3. 生成双向 `ReorderMap`（正向 + 反向映射）存入归档

### ReorderMap 编码

重排映射使用 **ZigZag 增量 + varint** 编码（`src/reorder_map.rs`）：

1. 计算相邻序号的差值（.delta）
2. ZigZag 编码处理负数：`(n << 1) ^ (n >> 63)`
3. 无符号 varint 编码实现紧凑存储

## SCM 质量值压缩

质量分数使用**统计上下文模型 (SCM)** + 算术编码压缩。

实现位于 `src/algo/quality_compressor.rs`。

### 上下文模型

| 读段类型 | 上下文阶数 | 上下文来源 |
|----------|------------|------------|
| 短/中等读段 | 二阶模型 | 前 2 个质量值 |
| 长读段 | 一阶模型 | 前 1 个质量值 |

### 算术编码

- 每个上下文维护自适应频率模型
- 当累计频率超过阈值时进行频率重缩放
- 32 位精度算术编码器/解码器

### 质量模式

| 模式 | 描述 | 压缩比影响 |
|------|------|------------|
| 无损模式 | 精确保留所有质量值 | 基准 |
| Illumina8Bin | 量化为 8 个代表值 (2,6,15,22,27,33,37,40) | 提升约 30% |
| 丢弃模式 | 全部替换为 `!` (Phred 0) | 最高压缩 |

## ID 压缩

读段标识符根据 ID 处理模式进行压缩。

实现位于 `src/algo/id_compressor.rs`。

| 模式 | 描述 | 典型场景 |
|------|------|----------|
| 精确模式 | 完整保留原始 ID | 需要精确 ID 匹配 |
| 去注释模式 | 移除第一个空格后的内容 | 常规用途 |
| 丢弃模式 | 替换为序号 `@read_N` | 最大压缩 |

ID 以换行符分隔拼接后，使用 Zstd 压缩为单个数据流。

---

## Zstd 编解码器（中等/长读段）

对于长度超过 300bp 的读段，序列使用长度前缀编码后 Zstd 压缩：

```
[u16: 读段长度][读段序列字节] × N 读段
```

Zstd 压缩级别可配置（1-19，默认 3）。

## 配对端 (PE) 优化

配对端数据利用 R1/R2 序列的反向互补关系进行优化压缩。

实现位于 `src/algo/pe_optimizer.rs`。

### 算法

1. 对 R2 序列取反向互补
2. 与 R1 序列比较相似度
3. 若相似度超过阈值，仅存储差异位置和碱基（增量编码）
4. 质量值差异类似处理

### PE 存储布局

| 布局 | 存储方式 | 描述 |
|------|----------|------|
| 交错式 | R1, R2, R1, R2, ... | 交替存储读段对 |
| 连续式 | R1, R1, ..., R2, R2, ... | 先存所有 R1 再存所有 R2 |
