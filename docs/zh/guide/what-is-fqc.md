# 什么是 fqc？

**fqc** (fq-compressor-rust) 是一个用 Rust 编写的高性能 FASTQ 压缩器。它实现了 [**ABC（基于对齐的压缩）**](https://github.com/shubhamchandak94/Spring) 算法用于短读段，以及 Zstd 用于中长读段。

## 为什么选择 fqc？

### 🚀 性能

- **压缩速度**: ~10 MB/s（默认），~12 MB/s（流水线模式）
- **解压速度**: ~55 MB/s（默认），~60 MB/s（流水线模式）
- **压缩比**: Illumina 配对端数据可达 3.9x

### 🧬 智能算法

| 读段长度 | 算法 | 压缩策略 |
|---------|------|---------|
| 短读段 (< 300bp) | **ABC** | 共识序列 + 增量编码 + 全局重排序 |
| 中等读段 (300bp – 10kbp) | **Zstd** | 直接压缩 + SCM 二阶模型 |
| 长读段 (> 10kbp) | **Zstd** | 直接压缩 + SCM 一阶模型 |

### 💾 质量压缩

统计上下文模型 (SCM) 配合算术编码：
- 短/中等读段使用**二阶上下文**
- 长读段使用**一阶上下文**
- 三种模式：`无损`、`illumina8bin`、`丢弃`

## 核心特性

- **随机访问**: 块索引格式支持部分解压
- **流式模式**: 从标准输入进行低内存压缩
- **流水线模式**: 3 阶段并行处理，带背压
- **配对端支持**: 交错或分离文件
- **压缩输入**: 透明解压 `.gz`、`.bz2`、`.xz`、`.zst`
- **内存预算**: 自动检测系统内存，动态分块

## 兼容性

fqc 与原 [C++ fq-compressor](https://github.com/LessUp/fq-compressor) 共享 `.fqc` 归档格式，确保两种实现之间的交叉兼容性。
